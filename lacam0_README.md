## 🎯 项目概述

LaCAM（Lazy Constraints for Multi-agent pathfinding）是一个高效的多智能体路径规划框架。本项目提供了完整的测试环境、工具链和可视化界面。

### ✨ 主要特点
- **真实算法**：基于官方LaCAM实现
- **完整工具链**：地图创建、场景生成、测试验证
- **可视化界面**：图形化参数设置和结果查看
- **多种测试场景**：从5个到2000个智能体的完整测试谱系
- **错误预防**：智能参数限制和错误处理

## 🗺️ 数据文件说明

### 地图文件 (`.map`)
- **作用**：定义环境的静态结构
- **格式**：octile格式，`.`可通行，`@`障碍物
- **示例**：
  ```
  type octile
  height 32
  width 32
  map
  ................................
  ..@.....@.....@.....@.....@.....
  ................................
  ```

#### 地图文件格式详细说明
MAPF Visualizer 使用 `.octile` 格式的地图文件，常见格式如下：

```
type octile
height 16
width 16
map
................
................
................
................
................
................
................
................
................
................
................
................
................
................
................
................
```

**字段说明：**
- `type octile`: 地图类型为八方向移动
- `height`: 地图高度（行数）
- `width`: 地图宽度（列数）
- `map`: 地图数据开始标记

**地图字符含义：**
- `.`: 可通过的空格
- `@`: 障碍物/墙壁
- `T`: 树木（可能不可通过）
- `S`: 水域（不可通过）
- `W`: 水域（不可通过）

**文件使用流程：**
1. **地图文件**：定义环境布局和障碍物位置
2. **场景文件**：定义智能体的起始和目标位置
3. **LACAM求解器**：读取场景文件，计算每个智能体的路径
4. **解决方案文件**：LACAM输出的路径规划结果
5. **MAPF Visualizer**：可视化解决方案文件中的路径

### 场景文件 (`.scen`)
- **作用**：定义具体任务（智能体起始和目标位置）
- **格式**：每行一个智能体，包含起始和目标坐标
- **示例**：
  ```
  version 1
  1	random-32-32-10.map	32	32	0	0	31	31	44.0
  2	random-32-32-10.map	32	32	0	31	31	0	44.0
  ```

#### 场景文件格式详细说明
场景文件定义了多智能体路径规划（MAPF）问题的具体实例，格式如下：

```
version 1
1 narrow-corridor-16-16.map 16 16 8 3 12 2 5.0
2 narrow-corridor-16-16.map 16 16 12 6 12 5 1.0
3 narrow-corridor-16-16.map 16 16 3 5 9 3 8.0
4 narrow-corridor-16-16.map 16 16 7 3 6 3 1.0
5 narrow-corridor-16-16.map 16 16 12 11 3 13 11.0
```

**各列含义详解：**

1. **智能体ID** - 智能体的唯一编号（1, 2, 3, ...）
2. **地图文件名** - 使用的地图文件名称（如 `narrow-corridor-16-16.map`）
3. **地图宽度** - 地图的列数（单位：格子）
4. **地图高度** - 地图的行数（单位：格子）
5. **起始位置X坐标** - 智能体起始位置的列索引（通常从0开始）
6. **起始位置Y坐标** - 智能体起始位置的行索引（通常从0开始）
7. **目标位置X坐标** - 智能体目标位置的列索引
8. **目标位置Y坐标** - 智能体目标位置的行索引
9. **最优路径长度** - 理论最优路径的曼哈顿距离或实际计算的最优解长度

**坐标系统说明：**
- X坐标：列索引，从左到右递增
- Y坐标：行索引，从上到下递增
- 原点(0,0)通常位于地图左上角

**示例解析：**
以第一行为例：
```
1 narrow-corridor-16-16.map 16 16 8 3 12 2 5.0
```
- **智能体1** 在 **16x16的狭窄走廊地图** 上
- **起始位置**：(8, 3) - 第8列，第3行
- **目标位置**：(12, 2) - 第12列，第2行
- **最优路径长度**：5.0（可能需要移动5步到达目标）

## 🔧 故障排除手册

### 问题1：段错误（返回码139）
**症状**：`段错误（核心已转储）`
**原因**：
1. 内存访问越界（算法bug）
2. 孤立单元格导致算法崩溃
3. 内存不足

**解决**：
```bash
# 减少智能体数量
./run_lacam -N 200

# 修复孤立单元格（已自动处理）
python3 tools/fix_all_isolated_cells.py

# 检查内存使用
free -h
```

### 问题2：invalid N错误
**症状**：`invalid N, check instance`
**原因**：智能体数量超过场景文件容量

**解决**：
```bash
# 查看文件最大容量
./run_lacam --list-files

# 减少智能体数量
./run_lacam -N 400  # 改为400或更少
```

### 问题3：找不到可执行文件
**症状**：`错误: 找不到 build/main 程序`
**解决**：
```bash
cd lacam0
make -C build -j4
```

### 问题4：GUI启动失败
**症状**：`错误: 找不到 Tkinter`
**解决**：
```bash
# 安装Tkinter
sudo apt install python3-tk

# 验证安装
python3 -c "import tkinter; print('Tkinter可用')"
```

### 问题5：运行时间太长
**症状**：算法长时间不结束
**解决**：
```bash
# 增加时间限制
./run_lacam -t 30

# 减少智能体数量
./run_lacam -N 200

# 使用更简单的地图
./run_lacam -s random-16-16-10-30.scen -m random-16-16-10.map
```

## 📈 性能优化建议

### 1. 智能体数量选择
| 测试目的 | 推荐数量 | 预计时间 |
|---------|----------|----------|
| 快速验证 | 5-30 | < 1秒 |
| 功能测试 | 50-100 | 1-3秒 |
| 性能测试 | 200-400 | 5-15秒 |
| 压力测试 | 500+ | 15-60秒+ |

### 2. 详细程度设置
- **0**：只显示基本结果（最快）
- **1**：显示进度信息（推荐）
- **2**：显示详细搜索过程
- **3**：显示所有调试信息（最慢）

### 3. 时间限制设置
- 快速测试：`-t 3`（3秒）
- 标准测试：`-t 10`（10秒）
- 压力测试：`-t 30`（30秒）
- 无限制：不设置 `-t` 参数

### 4. 内存使用优化
- 500智能体以下：通常 < 1GB
- 1000智能体：1-2GB
- 2000智能体：2-4GB
- 监控命令：`top` 或 `htop`
